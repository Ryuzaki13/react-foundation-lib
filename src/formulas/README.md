# Глобальный реестр формул таблицы

## Назначение

Реестр хранит набор клиентских формул, которые используются:

- в `CalculatedColumnsConfigurator` при выборе `formulaId`;
- в рантайме аналитической таблицы при вычислении `clientOnly` measure-колонок.

## Контракт формулы

```ts
(rowData: Record<string, unknown>, keys: string[]) => number;
```

- `rowData` — данные текущей строки таблицы;
- `keys` — список зависимостей (`formulaDependencies`) в заданном порядке.

## Структура

- `definitions/*` — по одной формуле на файл;
- `registry.ts` — построение реестра и проверка дубликатов `id`;
- `execute.ts` — безопасный запуск формулы с типизированным результатом.

## Экспериментальный вариант 2

Для сравнения добавлен `v2`:

- `v2/definitions/*` — формулы с контекстным API (`ctx.num(index)`, `ctx.value(index)`, `ctx.key(index)`);
- `v2/validate.ts` — отдельная валидация зависимостей для контракта `v2`;
- используется только как сравнительный сценарий и не заменяет основной runtime-путь.

## Как добавить новую формулу

1. Создать файл в `definitions/`.
2. Описать `id`, `name`, `description`, `fn` и при необходимости `examples`.
3. Подключить формулу в `definitions/index.ts`.
4. Добавить/обновить тесты в `registry.test.ts`.

Если в реестре появятся две формулы с одинаковым `id`, приложение получит явную ошибку при инициализации реестра.
